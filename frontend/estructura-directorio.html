# Estructura de Directorio

La estructura del proyecto es la siguiente, mostrada de manera resumida:

.
├── .nuxt
├── assets
│   ├── font-devinneorn
│   ├── font-icon
│   ├── font-lato
│   ├── imagenes
│   ├── scss
│   └── variables.scss
├── components
│   ├── admision
│   ├── elements
│   ├── AppActializarColegio.vue
│   ├── ...
│   ├── SiteCabecera.vue
│   └── ...
├── layouts
├── middleware
│   ├── autenticado.js
│   └── noAutenticado.js
├── mixis
├── pages
│   ├── dashboard
│   │   ├── articulo.vue
│   │   ├── ...
│   │   └── index.vue
│   ├── admision.vue
│   ├── index.vue
│   ├── login.vue
│   ├── propuesta.vue
│   └── sedes.vue
├── plugin
├── static
├── store
│   ├── autenticacion.js
│   ├── contenido.js
│   ├── enlacesNavegacion.js
│   └── mantenimiento.js
│ 
├── .env
├── ecosystem.config.js
└── 
└── vuetify.options.js
  • .nuxt: Es el directorio usado al momento de la construcción de la pagina web.
  • assets/font-devinneorn: Ubicación donde se encuentra la fuente de texto cima.
  • assets/font-icon: Ubicación donde se encuentra la fuente de iconos.
  • assets/font-lato: Ubicación donde se encuentra la fuente de texto lato.
  • assets/imagenes: Ubicación donde se encuentra las imagenes que usa la pagina, además de las imagenes que se usarán en la subpagina Admisión.
  • assets/scss: Ubicación donde se encuentran los archivos de estilos .scss.
  • assets/variables.scss: Archivo donde se importan los estilos.
  • components: Es el directorio donde se encuentran los componentes de la página web.
  • components/admision: Aquí se encuentran los componentes para la página Admisión.
  • components/elements: Aquí se encuentran los componentes generales de uso global.
  • components/AppActualizarColegio.vue: Archivo vue que se encarga de actualizar la información de la entidad colegio.
  • components/SiteCabecera.vue: Archivo vue que se encarga de mostrar la cabecera de la página web.

    Nota

    Los archivos con inicio App.. están encargados del mantenimiento del contenido y los archivos con inicio Site.. están encargados de mostrar el contenido.

  • layouts: Aquí se encuentran las estructuras donde se renderizarán las páginas que se encuentran en /pages.
  • middleware: Se encuentran los middleware que se utilizan para el login del dashboard.
  • mixns: Se encuentra el metodo para la bitacora.
  • pages: Se encuentran las páginas, como se nombren los archivos serán tomados en el router de la página web.
  • plugins: Se encuentran los plugins de la pagina, como el de Google Recaptcha.
  • statics: Se encuentran los recursos estáticos de la web.
  • store: Se encuentran los archivos encargados de administrar el estado de la web vuex.
  • store/autenticacion.js: Encargado de administrar si un usuario está logueado o no.
  • store/contenido.js: Encargado de administrar las peticiones de contenido de la web, aquí se encuentran las peticiones al servidor.
  • store/enlaces.js: Encargado de administrar los enlaces externos de la web.
  • store/mantenimiento.js: Encargado de administrar las peticiones al servidor encargados del mantenimiento de la web.
  • store/.env: Encargado de servir las variables de entorno de la web.
  • ecosystem.config.js: Encargado de configurar el script de producción.
  • vuetify.options.js: Encargado de configurar vuetify.
